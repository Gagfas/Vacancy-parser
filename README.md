# Парсер Вакансий 📊

Автоматизированный парсер вакансий с популярных сайтов поиска работы (hh.ru, SuperJob, Zarplata.ru) с функцией уведомлений о новых junior вакансиях.

## 🎯 Основные возможности

- **Парсинг вакансий** с нескольких источников:
  - hh.ru (с использованием Selenium)
  - SuperJob (через API)
  - Zarplata.ru (с использованием Selenium)

- **Автоматическое расписание** - периодическая проверка новых вакансий
- **Уведомления** - отправка уведомлений о новых junior вакансиях
- **База данных SQLite** - сохранение и отслеживание вакансий
- **Веб API** - FastAPI для доступа к данным через HTTP
- **Фильтрация** - черный список профессий/ключевых слов

## 📋 Требования

- Python 3.8+
- pip (менеджер пакетов)

## 🚀 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/yourusername/vacancy_parser.git
cd vacancy_parser
```

### 2. Создание виртуального окружения

```bash
python -m venv venv

# Для Windows:
venv\Scripts\activate

# Для Linux/Mac:
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка конфигурации

Создайте файл `.env` в корневой папке проекта:

```env
# API ключ SuperJob (получить на https://api.superjob.ru)
SJ_KEY=your_superjob_api_key

# Поисковый запрос
SEARCH_QUERY=Python разработчик

# Количество страниц для парсинга
MAX_PAGES=3

# Интервал проверки в часах
CHECK_HOURS=6

# Режим сырых данных (для тестирования парсер может улавливать не только вакансии по питону, а также любые, которые будут указаны в поисковом запросе. Если указать True, то будут искаться все вакансии, фильтрация по Junior вакансиям будет отключена, фильтры черного списка будут продолжать работать)
RAW_MODE=FALSE
```

## 🔧 Использование

### Запуск парсера один раз

```bash
python parser.py
```

### Запуск с автоматическим расписанием

```bash
python scheduler.py
```

Парсер будет проверять вакансии каждые N часов (указано в `CHECK_HOURS`).

### Запуск веб-сервера

```bash
python simple_web_api.py
```

API будет доступен по адресу `http://localhost:8000`

#### Доступные эндпоинты:

- `GET /` - главная страница
- `GET /stats` - статистика по вакансиям
- `GET /vacancies` - все вакансии
- `GET /juniors` - только junior вакансии
- `GET /unnotified` - неотправленные уведомления

### Парсер можно запустить в скрытом режиме, для этого используйте manager.bat

## 🗄️ База данных

### Таблица `vacancies`

Хранит информацию о вакансиях:
- `id` - уникальный идентификатор
- `title` - название позиции
- `link` - ссылка на вакансию
- `salary_from`, `salary_to` - диапазон зарплаты
- `currency` - валюта
- `description` - описание
- `platform` - источник (hh, sj, zarplata)
- `experience` - требуемый опыт
- `is_junior` - является ли junior позицией
- `notified` - отправлено ли уведомление
- `created_at`, `first_seen_at`, `notified_at` - временные метки

### Таблица `parse_log`

Логирование результатов парсинга:
- `parse_time` - время парсинга
- `hh_total`, `sj_total` - количество вакансий с каждого сайта
- `new_vacancies` - новые вакансии
- `new_juniors` - новые junior вакансии
- `status` - статус парсинга (success/error)

## 🐛 Решение проблем

### SuperJob API ключ не работает
- Получите ключ на https://api.superjob.ru/
- Убедитесь, что он указан в `.env` файле
- Проверьте, что ключ имеет необходимые права доступа

### Selenium ошибки
- Убедитесь, что Chrome/Firefox установлен
- `webdriver-manager` должен автоматически загрузить драйвер

### База данных заблокирована
- Закройте все открытые подключения
- Удалите `.db` файл и создайте новый
